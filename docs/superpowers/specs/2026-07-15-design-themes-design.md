# Вибір варіантів дизайну (тем оформлення)

**Дата:** 2026-07-15
**Статус:** затверджено, готово до плану реалізації

## Мета

Дати змогу перемикати «характер» презентації «Українці» між кількома
повноцінними темами оформлення (палітра + шрифти + стиль анімації/переходів +
ефекти на фото), не змінюючи контент і фотографії.

## Контекст

- Презентація — React + Babel, кадри керує рушій `SceneStage` (`animations-v2.jsx`),
  який перемальовує сцени щокадру (rAF). Це дозволяє живе перемикання: сцени
  читають глобал активної теми і підхоплюють зміну наступним кадром.
- Наразі палітра/шрифти/параметри анімації **захардкоджені** в `scenes.jsx`
  (константи `THEMES`, `SERIF`, `SANS`, `FADE`, `OUT`, коефіцієнти Ken Burns).
- Існує прихований перемикач light/dark у `tweaks-panel.jsx`, але він показується
  лише всередині рідного редактора (сигнал `__activate_edit_mode` від
  батьківського вікна). У standalone-презентації він **недоступний** — тому його
  для цієї задачі не використовуємо, а light/dark згортаємо (Класика лишається
  світлою). Окремих темних версій для всіх тем не робимо (YAGNI).

## Модель теми

Новий файл `src/themes.js` визначає реєстр `window.UK_THEMES` — масив обʼєктів:

```js
{
  id: 'classic',                    // стабільний ключ (для localStorage/URL)
  label: 'Класика',                 // назва в меню
  swatch: ['#F3EFE7','#26221B','#9A5B3B'], // 3 кольори для превʼю в меню
  palette: { bg, ink, muted, hair, accent },
  fonts:   { display, body, mono }, // CSS font-family рядки
  motion:  { photoFade, photoOut, kenBurns }, // числа: тривалості розчинення
                                              // (в одиницях progress) і сила
                                              // Ken Burns 0..1
  fx:      { filter: '', grain: false, vignette: false } // ефекти на фото
}
```

Перша тема в масиві — типова.

## Три теми

| | Класика (`classic`) | Модерн (`modern`) | Архів (`archive`) |
|---|---|---|---|
| Палітра | беж/графіт/теракота (поточна) | майже ч/б + синій акцент | тепла сепія |
| `fonts.display` | Playfair Display | Montserrat (гротеск, новий) | PT Mono |
| `fonts.body` | PT Sans | PT Sans | PT Sans |
| `motion` | мʼякі, повільні; kenBurns=1 | чіткі, швидші; kenBurns=0.4 | повільні; kenBurns=1 |
| `fx` | — | — | filter: sepia/contrast; grain+vignette |
| Нові ассети | — | 1 родина woff2 (Montserrat, локально, підмножини lat+cyr) | — |

Акцент Модерну — глибокий синій (узгоджено). Гротеск — Montserrat (надійний,
має кирилицю); беремо потрібні ваги підмножинами з Google Fonts, кладемо в
`assets/fonts/`, додаємо `@font-face` у `assets/fonts/fonts.css`.

## Зміни в коді

**`src/themes.js`** (новий) — реєстр тем; виставляє `window.UK_THEMES`.

**`src/scenes.jsx`** — рефактор споживання теми:
- `theme()` повертає активну тему цілком (палітра + fonts + motion + fx) на
  основі `window.__ukThemeId`; фолбек — перша тема.
- Прибрати модульні константи `SERIF`/`SANS`; заголовки/вступ/команда/титри
  сезонів беруть `T.fonts.display` / `T.fonts.body` / `T.fonts.mono`.
- `PhotoSequence` бере `FADE`/`OUT` із `T.motion.photoFade/photoOut`, силу
  Ken Burns множить на `T.motion.kenBurns`; якщо `T.fx.grain/vignette/filter` —
  накладає CSS-фільтр на `img` і оверлей-діви зерна/віньєтки поверх фото.
- Стан активної теми: `React.useState`, ініціалізація з
  `localStorage['uk_theme']` (фолбек — перша тема); при зміні — записати в
  `window.__ukThemeId` і `localStorage`.
- **Кнопка «Тема»** поруч зі «Звук» (той самий стиль, лівіше). Клік відкриває
  спливне меню: список тем, кожна — назва + свотч із 3 кольорів, активна
  позначена. Вибір міняє тему миттєво і закриває меню.

**`assets/fonts/fonts.css`** — додати `@font-face` для Montserrat (підмножини).

**`index.html`** — підключити `src/themes.js` перед `config.js`/boot (звичайний
`<script>`, як `config.js`).

## Живе перемикання (механізм)

`SceneStage` викликає рендер сцен щокадру. Сцени читають `window.__ukThemeId`
через `theme()`. Зміна стану в корені оновлює глобал → наступний кадр малюється
новою темою без перезапуску таймлайну. Це та сама схема, що вже працює для
поточного `window.__ukTheme` (light/dark).

## Перевірка

Тимчасовий harness (`_test_harness.html`, у `.gitignore`) з параметром теми:
рендер кожної сцени в кожній темі → скріншот у headless Chrome. Перевіряємо:
1. палітра застосовується (фон/чорнило/акцент);
2. шрифти вантажаться (Montserrat у Модерні, PT Mono в Архіві);
3. ефекти Архіву (сепія/зерно/віньєтка) видно на фото;
4. кнопка «Тема» відкриває меню, вибір міняє вигляд;
5. вибір зберігається в localStorage (повторний відкрив — та сама тема).

## Поза межами (YAGNI)

- Темні версії для всіх тем (лишаємо тільки світлу Класику).
- Стартовий екран-вибір (обрали кнопку в кутку).
- Редагування тем із UI, кастомні палітри користувача.
- Різні макети окремих кадрів (композиція між темами відрізняється лише
  дрібно через шрифти/палітру/ефекти, без окремих лейаутів).
